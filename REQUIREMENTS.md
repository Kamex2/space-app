# 3D太陽系エクスプローラー 要件定義書

作成: Claude Fable 5 / 実装担当: Claude Opus 4.8
プロジェクトルート: `C:\Claude\Fable5\solar-system-explorer`（このファイルがあるディレクトリ）

## 1. 概要

ブラウザで動く3D太陽系エクスプローラー。JPLの近似惑星軌道要素を使って任意の日付（1800〜2050年）の惑星配置を物理的に正しく再現し、さらに探査機を発射してスイングバイ（重力アシスト）軌道を設計して遊べるモードを持つ。

デモやAPIキーは不要。すべての計算はクライアント側で完結し、オフラインで動作すること。

## 2. 技術スタック（固定）

- Vite + TypeScript（テンプレート: `vanilla-ts`）。React等のUIフレームワークは使わない。
- Three.js（npm の `three` + `@types/three`）。OrbitControls、必要なら `three/examples/jsm` のポストプロセス（UnrealBloomPass）を使用可。
- 外部アセット（テクスチャ画像、CDN、Webフォント）は一切使わない。マテリアルは色・emissive・簡単なプロシージャル表現で構成する。
- UIラベルはすべて日本語。コード識別子は英語。

## 3. 機能要件

### F1. エフェメリス計算エンジン（最重要・精度の要）

- JPL "Approximate Positions of the Planets"（Standish, https://ssd.jpl.nasa.gov/planets/approx_pos.html）の Table 1（1800 AD – 2050 AD 有効）のケプラー軌道要素を使う。**必ずWebFetchで上記ページ（またはリンク先のaprx_pos_planets.pdf / p_elem_t1.txt）を取得し、8惑星（EM Barycenter含む）の a, e, I, L, ϖ, Ω とその世紀あたり変化率を正確に転記すること。** 記憶からの転記は誤植リスクがあるため不可。取得した数値はソース内の定数テーブル `src/data/planetElements.ts` に、出典URLコメント付きで格納する。
- 計算手順（JPL文書の手順に厳密に従う）:
  1. J2000からのユリウス世紀数 T を計算し、各要素を `elem = elem0 + rate * T` で更新
  2. 平均近点角 M = L − ϖ（[-180°, 180°] に正規化）、近日点引数 ω = ϖ − Ω
  3. ケプラー方程式 M = E − e·sin(E) をニュートン法で解く（許容誤差 1e-8 rad、初期値 E = M + e·sin(M)）
  4. 軌道面座標 → 黄道座標（J2000黄道・春分点基準）への回転変換（ω, I, Ω）
- 出力: 太陽中心・黄道座標系の位置 [AU]。速度も必要（スイングバイモードで使用）。速度は解析微分（dE/dt = n / (1 − e·cosE) 経由）で求める。数値微分でも可だが精度を落とさないこと。
- 対象天体: 水星・金星・地球（EM barycenter）・火星・木星・土星・天王星・海王星の8惑星。

### F2. 3Dシーン

- 太陽（emissive + 控えめなbloom）、8惑星、各惑星の軌道線（軌道要素から楕円を直接生成、現在日付の要素で描画）。
- 背景は星空（乱数配置のPoints、数千点）。
- 距離スケール切替: 「実スケール」と「圧縮スケール」（r' = r^0.6 など、外惑星が見える圧縮写像）をトグルできる。軌道線・惑星位置・探査機軌道すべてに一貫して同じ写像を適用すること。
- 惑星サイズは視認性のため誇張表示（デフォルト）。「実寸比」トグルで実比率にできる。
- 各惑星に日本語ラベル（CSS2DRendererまたはスプライトでカメラ正対）。遠い惑星でもラベルは読めること。
- カメラ: OrbitControls。惑星クリックでその惑星にフォーカスし追従。ダブルクリックまたは「太陽に戻る」ボタンで解除。

### F3. 時間制御

- 表示範囲: 1800-01-01 〜 2050-12-31（Table 1の有効範囲）。
- UI: 日付表示（YYYY年MM月DD日）、日付入力（`<input type="date">`）、再生/一時停止、速度スライダー（1日/秒 〜 1年/秒、逆再生も可）、「今日」ボタン。
- 再生中は惑星がなめらかに公転する（毎フレーム、エフェメリスをフル再計算して問題ない計算量のはず。60fps維持）。

### F4. 惑星情報パネル

- フォーカス中の惑星について: 太陽からの現在距離 [AU]、公転周期、離心率、軌道傾斜角、現在の日心黄経を表示。値はすべて適切な桁数に丸める。

### F5. スイングバイ設計モード（目玉機能）

- モード切替ボタンで通常モード⇔設計モード。
- 探査機の打ち上げ: 現在日付の地球位置から出発。ユーザーは以下をスライダーで指定:
  - 出発余剰速度 v∞: 0.5〜16 km/s
  - 方向: 黄道面内角度（地球公転速度ベクトル基準 −180°〜+180°）と面外角度（−45°〜+45°）
  - 初速 = 地球の公転速度ベクトル + v∞ベクトル
- 軌道伝播: 太陽+8惑星の重力を含むN体（探査機はテスト粒子）。積分はRK4固定ステップ（基本 0.5日、惑星から0.1 AU以内では 0.02日以下に細分）。単位系は AU・日で統一（換算: 1 km/s = 5.7755e-4 AU/day。GM値は §5 参照）。
- スライダーを動かすたびに、その場で最大15年分の予測軌道を再計算して線で描画する（Web Worker化は不要だが、60fpsを大きく損なう場合は計算を間引く/デバウンスする）。
- フライバイ検出: 惑星に 0.05 AU 以内まで接近したら軌道上にマーカーと「◯◯スイングバイ YYYY/MM/DD、通過距離、速度変化 Δv」を表示。太陽中心速度の増減が分かること。
- 「発射」ボタンで確定すると、時間再生に同期して探査機が軌道上を移動する。探査機にもラベルと軌跡を表示。
- 遊びの目標として、パネルに各外惑星への「最接近距離」記録を表示（例: 木星まで 0.03 AU まで接近!）。

### F6. UI/HUD

- 画面端に半透明ダークのHUDパネル（上: 日付と時間制御、右または左: モード・スケール切替と情報パネル、下: スイングバイ設計スライダー群）。CSSは手書きで良いがモダンで統一感のある見た目にする（フラット、過剰装飾なし）。
- 操作説明を初回に小さく表示（ドラッグで回転、ホイールでズーム、クリックでフォーカス）。

## 4. 非機能要件

- 60fps目標（フルHD、統合GPU想定）。ジオメトリ再生成を毎フレーム行わない（軌道線は日付変化が大きい時のみ更新、探査機予測線はパラメータ変更時のみ再計算）。
- `npm run build` がエラーゼロで通ること。`tsc` の型エラーもゼロ。
- モジュール分割: `src/data/`（軌道要素・物理定数）、`src/ephemeris/`（ケプラー計算）、`src/sim/`（RK4伝播・フライバイ検出）、`src/scene/`（Three.jsシーン構築）、`src/ui/`（HUD）、`src/main.ts`。

## 5. 物理定数（この値を使うこと）

- GM_sun = 2.9591220828559093e-4 AU³/day²
- 惑星のGMは太陽との質量比から算出: 水星 1/6023600, 金星 1/408523.71, 地球+月 1/328900.56, 火星 1/3098708, 木星 1/1047.3486, 土星 1/3497.898, 天王星 1/22902.98, 海王星 1/19412.24
- 1 AU = 1.495978707e8 km、1 km/s = 5.7755e-4 AU/day

## 6. 受け入れ検証(実装完了前に必ず実施し、結果を報告に含める)

1. `npm run build` 成功。
2. サニティテスト(`npx tsx` またはviteのnodeスクリプトで実行できる `scripts/sanity.ts` を作る):
   - 全惑星・全年代（1800/1900/2000/2050の各1月1日）で |r| が近日点距離〜遠日点距離の範囲内
   - 地球: 1月上旬に |r| ≈ 0.983 AU（±0.002）、7月上旬に ≈ 1.017 AU（±0.002）
   - 水星の公転周期 ≈ 88日（連続する近日点通過から確認、±1日）
   - ケプラー方程式ソルバが e=0.2, M=全域 で残差 < 1e-8
3. スイングバイ: v∞やRK4のエネルギー保存チェック（惑星接近がない2体区間で軌道エネルギーの相対誤差 < 1e-6）。

## 7. 実装しないもの（スコープ外）

- 月・小惑星・冥王星、テクスチャマッピング、実照度シミュレーション、セーブ機能、モバイル最適化、多言語切替。
